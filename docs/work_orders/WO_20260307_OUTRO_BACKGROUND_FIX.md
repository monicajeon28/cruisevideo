## . 문제 정의

### . 현상
- **파일** .  -
- **문제**  구간(초)에서 순수 검은색 배경 ((,,)) 사용
- **발생 조건** - 처리 실패 시  → 검은색 배경
- **발생 확률** 약 .% (초 / 초)

### . 사용자 경험 영향
 측면  영향 
------------
 **시각적 단절**  영상→검은색→로고 부자연스러운 단절 (브랜드 각인 -%) 
 **완주율**  검은색 배경에서 이탈 가능성 +% (초 도달 후 이탈) 
 **전문성**  "완성되지 않은 영상" 인상 (신뢰도 -%) 
 **브랜드 일관성**  전체 영상 톤과 불일치 (크루즈  여유/바다  검정)

## . 해결 방안 가지 비교

### 옵션  마지막 영상 재사용 (  )

**설명** 의 마지막 클립을 희미하게 재사용 (현재 로직 강화)

**장점**
-  구현 간단 (기존 로직 개선만)
-  추가 에셋 불필요
-  영상 연속성 유지 (자연스러운 흐름)

**단점**
-  마지막 영상이 에 부적합한 경우 (예 텍스트 오버레이, 움직임 많은 영상)
-  반복 느낌 (초 이후 같은 영상)

**구현 예시**

     
    #   마지막 클립 강제 재사용
      -''  (-, )  -
      .(.(.))
      .(, (, .))
      .()

### 옵션   전용 이미지 풀 (  )

**설명** 크루즈 테마  전용 이미지 풀 생성 (/////)

**장점**
-  전문적 마무리 (브랜드 일관성 +%)
-  다양성 제공 (매번 다른  배경)
-  테마 통제 가능 (// 분위기)

**단점**
-  에셋 추가 작업 필요 (이미지 선별/분류)
-  구현 복잡도 증가 (. 연동)
-  유지보수 비용 (이미지 풀 관리)

**구현 예시**

# .에 추가
   ( 
    "/////.",
    "/////.",
    "/////."
)

# .에서 사용
   
      .(..)
      ().()
      .(.(.))  # 희미하게

### 옵션  다단계  시스템 (- )

**설명** 단계  (마지막 영상 →  이미지 풀 → 검은색)

**장점**
-  튼튼한 시스템 (.% 성공률)
-  최선의  (각 단계별 최적 선택)
-  검은색 배경 최소화 (.% 미만)

**단점**
-  구현 복잡도 최고
-  테스트 케이스 많음 (단계 모두 검증 필요)
-  코드 가독성 저하 가능

## . 성공 기준

### . 정량적 지표

 지표  목표  측정 방법 
-----------------------
   성공률  **≥ %**  로그 분석 (편 렌더링) 
   사용률  **%**  로그 " " 카운트 
   (검은색)  ** .%**  로그 "  " 카운트 
 브랜드 각인  **+%**  / 테스트 (검은색   ) 

### . 정성적 지표

-    배경이 영상 톤과 일치 (크루즈/바다/여유)
-   로고가 배경과 대비 명확 (가독성 %)
-   영상 연속성 자연스러움 (단절 느낌 %)
-   로그 메시지 명확 ( // 구분)

## . 참고 자료

### . 관련 파일
- .  - ( 배경 로직)
- /.  - ( 설정)
- ///// (이미지 풀 경로)

### . 관련 
- ** -** 이미지 기반 자막 렌더링 (초)
- **  -**  .→.초 단축
- **-- --** 로고 중앙 배치 + 검은색 배경

### .  
- **검은색 배경 의도** 로고 강조 (흰색 로고 + 검은색 배경  대비 %)
- **  우선순위** / 테마 (크루즈  여유/바다 연상)
- **   .** 로고 가시성 유지 (배경 %  로고 %)
