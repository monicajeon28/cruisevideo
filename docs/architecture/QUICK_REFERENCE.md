##  자주 묻는 질문 ()

###   대신  를 선택한 이유

**** 렌더링 성능 저하 (+%) + / 불필요
- 현재 단일 프로세스로 충분 (네트워크 오버헤드 불필요)
- 배포 복잡도 증가 (,  관리 비용)
- 점진적 마이그레이션 불가능 (전체 재작성 필요)

###  기존 파이프라인은 어떻게 되나요

****    (점진적 교체)
-    % +  / 테스트
-    % +  %
-    % (래퍼만 유지) +  %

###  렌더링 시간이 느려지지 않나요

**** 성능 오버헤드  초 (.%)
- 모듈 간 함수 호출 오버헤드 무시 가능
-  초 유지 (초 이내)
- / 테스트로 검증 (허용 기준 % 이내)
