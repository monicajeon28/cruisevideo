## .  영상 생성  비교 분석

### -. 현재 프로젝트에 이미 존재하는 엔진

프로젝트 내에 이미 개의  영상 생성 엔진이 구현되어 있다.

 파일    모델  해상도  길이  비용 
-------------------------------------
 /.  .     /    또는   초  $. () / $. () 
 /.   .  .  /  초 또는 초  $./초 (- 무료) 
 /.   ( .)     ()  초  $. 
 /.   ( .)  /-.    초  $. 

### -. 전체  후보 비교 (년 월 기준)

   세로형   최대 해상도  최대 길이  비용/초 클립      크루즈 씬 품질 
---------------------------------------------------------------------------------
 ** .** ()      초  $.-.       
 ** .**   (에서 지원)    초  $.-.       
 **.  **   ( 지원)    초  $. ()       
 ** -  **      초  $.       
 ** .**      초  $.      + 
 **   .**      초  $.       
 ****  제한적    초  $.+     (제한)   
 ** ()**      초  $.       

### -. 크루즈 숏폼 최적  추천

**순위 .   ** (이미 구현 완료)
- 이유  세로형 네이티브 지원, 초 길이, $./클립, 프로젝트에 이미 구현됨
- /.가 이미 완전한  구현 (생성-폴링-다운로드)
-  모드로 프롬프트만으로 크루즈 씬 생성 가능

**순위  .** (이미 테스트 완료)
- 이유  품질 우수, $./초
- /.에 기본 구현 존재 ( )

**순위  .** (이미 구현 완료)
- 이유 - 무료 생성 가능, 비용 최적화에 유리
- /.가 완전한  구현

**비용 예측 (일 개 영상 기준)**
- 도입부 초  개 클립  초    호출
- .    $.  $./영상, 일 개  **$/일**
-  - **$/일** (무료),  시 $./영상
-    $.  $./영상, 일 개  **$/일**
- 캐시 히트율 % 적용 시 **$-/일** (. 기준)

### -. 캐시 히트율 극대화 전략


. 프롬프트 정규화 ( )
   - 동의어 통합 "바다"  ""  ""  "해양"
   - 불용어 제거 "", "", "" 등
   - 키워드 정렬 를 정렬된 키워드 집합으로 변환 후 해시
   - 예상 히트율 기본 % - 정규화 후 %+

. 씬 카테고리 기반 검색
   - 정확한 프롬프트 매칭 실패 시 같은 카테고리 내 대체 클립 사용
   -  카테고리에 개 클립이 있으면 랜덤 선택 (반복 방지)
   - 예상 히트율 %+ (충분한 클립 축적 후)

. 계절/시간대 태그
   -  필드에 "", "", "" 등 추가
   - 스크립트의 분위기와 매칭하여 더 적합한 클립 선택

.  + 품질 기반 캐시 정리
   - 최대 용량  (약 ,개 클립)
   - 정리 기준 가 이고 일 이상 미사용 클립 삭제
   - 가  미만인 클립 우선 삭제

## .  (  )

### -  영상 생성  선택

**상태** 제안됨

**컨텍스트** 도입부 초를  생성 영상으로 교체하여 스톡 영상 의존도를 낮추고 시각적 품질을 향상시키려 한다. 프로젝트에 이미 ., ,  개  엔진이 구현되어 있다.

**결정** .   를 순위로,  .과  .을 폴백으로 사용하는 - 구조를 채택한다.

**결과**
- 장점 단일  장애 시 자동 폴백, 비용 최적화 ( - 무료 활용), 기존 코드 재활용
- 단점 개  관리 복잡도 증가,  응답 형식 통일 필요
- 대안  - (품질 최고이나 비용 배),  (세로형 제한, 비용 최고)

### - 캐시 저장소 선택 (  )

**상태** 제안됨

**컨텍스트** 생성된  클립의 메타데이터를 저장하고 효율적으로 조회해야 한다. 기존 프로젝트에서는 .으로  캐시를 관리하고 있다.

**결정** 를 메타데이터 저장소로 사용한다.

**결과**
- 장점 복합 쿼리 (카테고리+품질+사용빈도), 인덱스 기반 () 조회,  보장, + 레코드에서도 성능 저하 없음
- 단점  대비 코드 복잡도 증가, 별도 마이그레이션 필요
- 대안  파일 (기존 패턴과 일관, 개 미만이면 충분),  (과도한 인프라)

### - 기존 파이프라인과의 통합 방식

**상태** 제안됨

**컨텍스트** 기존  메서드가 /를 순차적으로 로드하는 구조이다.  도입부는 기존 (초)을 초로 확장하는 형태이다.

**결정**  내에서  플래그 기반으로 분기하여,  도입부 개 클립을 기존  리스트에 삽입한다. 스크립트 의  구조는 변경하지 않는다.

**결과**
- 장점 기존 코드 최소 변경, 로 즉시 기존 모드 복귀 가능, 자막//페르소나 로직 변경 불필요
- 단점 세그먼트 인덱스 정합성 관리 필요 (개  클립  기존 개 )
- 대안 스크립트  구조 자체를 변경 (큰 파괴적 변경), 별도 렌더링 후  (이중 인코딩 품질 저하)
