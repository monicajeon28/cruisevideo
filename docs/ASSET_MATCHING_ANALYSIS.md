##  

현재  는 **단계 우선순위 체계**로 작동하지만, ** 구간(-.초)**에서 기항지 비주얼 대신 선박/일반 이미지가 선택되는 문제가 있습니다.

### 핵심 문제점 가지

 문제  현재 상태  기대 상태  영향 
----------------------------------
 **-**  구간 매칭 우선순위   (선박/파티) 우선  기항지 비주얼 우선  도파민 자극 -% 
 **-**  영한 매핑 불완전  개 중 개만 매핑  개 전체 매핑  매칭률 -% 
 **-**  과다 사용  % 이상    % 미만   품질 저하

## . 매칭 우선순위 체계 분석

### 현재 우선순위 (단계)


# () 함수 내 우선순위 순서

순위 -- 기항지 폴더 우선 (/ 섹션만)
  - 조건   "", ""    "", ""
  - 폴더 기항지풍경, 그리스 산토리니, 그리스 미코노스, 크로아티아 두브로브니크, 알래스카, 항구, 크루즈정보사진정리, 유럽크루즈
  - 점수 기항지명 직접 매칭 +점, 파일명 매칭 +점
  - 최소 점수 점 (--)
  - **문제**  섹션이 제외됨!

순위 - 후기 이미지 우선 (// 섹션만)
  - 조건   "", "", "", ""
  - 폴더 /후기
  - 점수 고정 점
  - 상태  정상 작동

순위 -  직접 
  -  (개 매핑) +  (개 매핑)
  - 보너스 +점 (  → 하향)
  - **문제** 에서도 작동하지만 보너스가 낮아   에셋에 밀림

순위  매핑 기반 키워드 매칭
  -  . (,개)
  -  . (수천 개)
  - () 점수 계산 (-)
  -  보너스 +점 (-)
  -  보너스 +점 ( 매칭 시)
  -  보너스 +점 (같은 시리즈 에셋)
  -  부스트 +점 (/)

순위  (/)
  -   감지된 카테고리 내  선택
  -    카테고리
  - **문제** % 이상 사용 시 매칭 로직 실패를 의미

## . 점수 계산 방식 분석

### () 로직


#  - 점수 계산 공식

  계산
  - 기항지 키워드 정확 매칭 +점 (--)
  - 구체적 키워드 정확 매칭 +점
  - 공통 키워드 매칭 (크루즈/바다/여행) +점 (-)
  - 부분 매칭 () +점
  - 파일명 매칭 +점
  - 퍼지 매칭 (  .) +점

섹션별 부스트 ( -)
  -  섹션 + 기항지 검색
      . *   .배
  -  섹션 + 기항지 검색
      . *   .배
  -  섹션 + 기항지 검색
      . *   .배

보너스 점수
  -  보너스 .() *  (최대 점)
  -   보너스 +점 ( 매칭 시)
  -  보너스 +점 (같은 시리즈 에셋)
  -   부스트 +점 (/)
  -   보너스 +점 ( → 하향)


###  **문제점**   보너스  기항지 부스트


# 시나리오  구간에서 산토리니 키워드 검색

  산토리니 이미지
  -    (기항지 매칭) +  (부분) +  (파일명)  점
  -    *   점 (최종 점으로 )
  -  +점
  - 총점 점 ()

   벨리시마 골든파티 이미지
  -    (공통 '크루즈') +  (부분)  점
  -   +점
  -  +점
  - 총점 점

→ 기항지 이미지가 이긴다!


**그런데 왜 문제가 발생할까**

→ **순위 기항지 폴더 검색이 에서 실행되지 않기 때문**

## .  체계 분석

###  단계


# () 내  흐름

차 기항지 폴더 우선 검색 (/만)
  → 실패 시 차로 이동

차 파일명 기반 기항지 검색 ()
  → 실패 시 차로 이동

차 후기 이미지 우선 (//만)
  → 해당 없으면 차로 이동

차  직접 
  → 실패 시 차로 이동

차  매핑 키워드 매칭
  → 실패 시 차로 이동

차   (감지된 카테고리 내  선택)
  → 실패 시 차로 이동

차   ( 카테고리)
  → 실패 시  반환
