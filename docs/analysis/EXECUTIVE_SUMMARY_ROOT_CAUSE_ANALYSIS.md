## 근본 원인 가지 패턴

### 패턴  작업 스코프 불일치 ( 개 누락)

**  분석**

  왜 , , 의 이 없는가
→  가 만 변환, 와 는 미생성

  왜 와 를 생성하지 않았는가
→ 작업 지시서에 "만 생성" 명시

  왜 작업 지시서에 와 가 누락되었는가
→   산출물 (종)    요구사항 ( 시스템) 인계 누락

  왜  변환이 일부만 수행되었는가
→  .에 " 종 →  변환" 명시 없음

  (근본 원인) 왜  .에 명시되지 않았는가
→  간 작업 인계 시 마이그레이션 계획 부재


**해결책**    (분) + 자동 검증 스크립트 (분)

### 패턴  수동 변수 정의 방식 (,  미정의)

**  분석**

  왜 와 가 .에 없는가
→  가 변수 작성 시 기존 템플릿 스캔하지 않음

  왜 템플릿 스캔을 하지 않았는가
→ - 설계 (신규 시스템 우선, 역호환 검증 나중)

  왜 역호환 검증이 누락되었는가
→ 작업 지시서에 "개 변수 정의"만 명시, "템플릿 스캔" 단계 없음

  왜 템플릿 스캔 단계가 없는가
→ 신규 시스템 구축에만 집중 (기존 파일 통합 계획 부재)

  (근본 원인) 왜 기존 파일 통합이 누락되었는가
→ 수동 변수 정의 방식 + 자동화된 변수 탐지 시스템 부재


**해결책** 자동 변수 추출 스크립트 (분) ← **이미 구현됨**

**예방 확률** %

### 패턴  복잡도 예측 실패 ( 등 개 기능 미구현)

**  분석**

  왜 , , 캐시가 미구현인가
→ .가 에 집중

  왜 에 집중했는가
→ "시간 구현" 제약

  왜 시간 제약이 있는가
→  . 타임라인에서 .를 시간으로 과소평가

  왜 과소평가했는가
→ 초기 예상 "단순 치환 엔진" → 실제 "검증+캐싱+동적생성"

  (근본 원인) 왜 복잡도를 잘못 예측했는가
→ 요구사항 분석 단계에서 숨겨진 복잡도 미발견


**실제 복잡도**
- 예상 시간 (.() 치환)
- 실제 시간 (검증  + 평탄화  + 재귀  + 캐싱  + 섹션  + 에러  + 테스트 )

**해결책** 기능 분해 + 우선순위 설정 (//) →  로 연기
