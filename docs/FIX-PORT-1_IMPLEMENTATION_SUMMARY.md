# -- 구현 완료 보고서

## 목표
등급 점수 .점 → .점 (+.점)
-   ./점 → ./점 (만점 달성)
- 기항지 비주얼 .초 → 초 (초 목표 달성)

## 문제 분석

###  ( .)
- 기항지 비주얼 .초 / 초 (% 달성)
-  -.초 (-%)
- 원인  초 중 .초만 기항지 → % 비율
- (초) + (.초) + (초)  .초는 브랜드/로고 영상

### 근본 원인
. **대본 생성** 기항지 밀도 낮음 (추상적 표현 다수)
. **키워드 추출** 기항지명 우선순위 낮음
. **이미지 매칭** 기항지 폴더 우선순위 없음
. **검증 로직**  전체를 카운트 (// 포함)

## 구현 전략

### . 대본 생성 단계 기항지 밀도 강화 

**파일** .
**위치**  프롬프트 ( -)

**변경 내용**

 -- 기항지 중심 대본 작성 (필수) 

! **등급 필수 조건 기항지 비주얼 초 이상 ( % 이상)**

**.  세그먼트 개 중 최소 개는 기항지 정보 (.% 이상)**
   - 선박 시설은 개 세그먼트만 배치 (뷔페, 수영장, 객실)
   - 나머지 개는 기항지 중심 (도시명 + 명소 + 특징)

**. 세그먼트 분배 예시 ()**
     - 기항지  (도시 + 랜드마크)
      선박 시설
     - 기항지 , ,  (관광 명소)
      기항지  (항구 풍경)
      기항지 요약

**. 나쁜 예 () - 기항지 부족**
     - 선박 객실, 뷔페, 수영장 (기항지 없음)
      "여러 나라를 방문합니다" (추상적, 구체적 기항지명 없음)


**효과**
-  세그먼트 개 중 개(.%) → 기항지 집중
- 구체적 기항지명 + 명소 조합 필수
- 추상적 표현 금지 ("여러 나라" → "산토리니, 미코노스, 아테네")

### . 키워드 추출 강화 기항지명 우선 

**파일** .
**위치** () 메서드 ( -)

**변경 내용**

 (,  ) - 
    """
    고유명사 추출 (-- 기항지명 배 가중치)
    """
      
      .()

       
             .()  
            # -- 기항지명은 번 추가 (가중치 배)
               
                .( * )
                .("-- 기항지명 배 부스트 ")
            
                .()

     


**추가**  연관 키워드 확장 ( -)

# -- 기항지명인 경우  연관 키워드 추가
       
        # 상위 개
       
          .(, )
            .((
                ,
                .,
                "",
                ,
                ,
            ))


**효과**
- 기항지명 가중치 배 → 키워드 추출 시 최우선
-  연관 키워드 자동 추가 (예 "산토리니" → "", "그리스", "에게해", "블루돔", "일몰")
