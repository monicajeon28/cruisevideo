## 아키텍처 결정 기록 ()

### - 모놀리식  마이크로서비스 아키텍처

**상태** 승인됨

**컨텍스트**
- 초기  단계로 빠른 개발 및 배포 필요
- 개 컴포넌트 간 강한 결합도 (순차 처리)
- 단일 서버 환경 (로컬 에셋 의존성)

**결정**
  아키텍처 채택
- 단일  애플리케이션
- 컴포넌트는 명확한 인터페이스로 분리
- 향후 마이크로서비스 전환 가능 구조

**결과**

장점
- 빠른 개발 속도
- 간단한 배포 및 운영
- 낮은 인프라 비용
- 트랜잭션 관리 용이

단점
- 수평 확장 제한
- 컴포넌트별 독립 배포 불가
- 장애 격리 어려움

**대안**
. 마이크로서비스 초기 오버헤드 너무 큼
.   처리에 부적합 (실행 시간 제한)

### -     

**상태** 승인됨

**컨텍스트**
- 영상 합성 엔진 선택 필요
- 요구사항 고품질 인코딩, 빠른 처리, 안정성

**결정**
   사용

**결과**

장점
- 업계 표준, 검증된 안정성
- 하드웨어 가속 지원 ()
- 복잡한 필터 체인 지원
- 최고 품질의 인코딩

단점
-  호출 오버헤드
- 에러 핸들링 복잡
-  통합 코드 필요

**대안**
.  메모리 사용량 과다, 느린 처리
.  비디오 편집 기능 제한적

### - 동기  비동기 처리

**상태** 승인됨

**컨텍스트**
- 영상 생성은 장시간 작업 (-초)
- 사용자 경험 실시간 진행 상황 필요

**결정**
비동기   +  실시간 알림

**결과**

장점
- 즉각적인 응답 ( 반환)
- 실시간 진행률 알림
- 서버 부하 분산 가능

단점
- 구현 복잡도 증가
-  저장소 필요 ( 권장)

**대안**
. 동기 처리 타임아웃 위험, 사용자 경험 저하
. 폴링 네트워크 오버헤드 과다

### -     

**상태** 승인됨

**컨텍스트**
- 개 비디오 (약 )
- 개 이미지 (약 )
- 빠른 읽기 속도 필요

**결정**
로컬  저장 (//)

**결과**

장점
- 빠른 / 속도
- 네트워크 비용 없음
- 오프라인 작업 가능

단점
- 확장성 제한
- 백업 관리 필요
- 멀티 서버 배포 어려움

**대안**
.   네트워크 레이턴시, 비용 증가
.  / 속도 느림

### -      

**상태** 승인됨

**컨텍스트**
- 한국어 자연스러운 감정 표현 필요
- 다양한 페르소나 음성 (, , )

**결정**
  

**결과**

장점
- 한국어 최적화
- 감정 표현 우수 ( 지원)
- 페르소나 다양성

단점
-  비용
- 외부 의존성

**대안**
.  한국어 품질 낮음
.   감정 표현 제한적
