# 범용 주제별 리소스 자동 수집 시스템 (   )

**목표** 어떤 주제든 자동으로 **%  **를 달성하는 완전 자동화 시스템

---

##  시스템 개요

### 문제 정의
기존 시스템은 **후쿠오카에만 특화**되어 있어 다른 주제(오사카, 발리, 타이페이 등)에서는 리소스 부족 문제 발생


#  기존 문제점
  
    "  ",  # 하드코딩!
    "  ",       # 후쿠오카 전용!

  ""  # 고정 폴더명!


### 해결 방안
** 기반 주제 추출 + 동적 키워드 생성 + 자동 다운로드** 파이프라인


제목 입력 → 주제 추출 → 키워드 생성 → 리소스 다운로드 → % 검증
   ↓           ↓            ↓              ↓              ↓
"오사카 맛집"  오사카+맛집  " "   /   달성

##  성능 및 검증

### %   달성 전략

**목표** 영상당 필요한 푸티지 -개 중 % (-개)를 로컬에서 조달

**전략**
. **키워드 다양성 확보** 일반() + 구체적() + 맥락() + 감성()  개 키워드
. **키워드당 개 다운로드**  키워드  개  개 영상
. **중복 제거 후** 약 -개 유니크 영상
. **품질 필터링 후** 약 -개 사용 가능 영상
. **결과** -개 필요 → -개 확보  **-% 달성**

**검증 코드**

##  트러블슈팅

### 문제  키워드 매칭 실패

**증상** 생성된 키워드로 검색해도 영상이 없음

**원인** /에 해당 키워드 리소스 부족

**해결책**
. 더 일반적인 키워드로 확장
. 유사 도시 키워드 활용 (예 "" → " ")
. 카테고리 키워드 추가 (예 " " → " ")


# . 수정
  
    "" " ", "", " ",
    "" " ", "", " ",
    # ...



---

### 문제  다운로드 속도 느림

**증상** 개 영상 다운로드에 분 이상 소요

**원인** 병렬 처리 워커 수 부족, 네트워크 대역폭 제한

**해결책**
. 워커 수 증가 --  (기본  → )
. 키워드 우선순위 적용 고품질 키워드만 우선 다운로드
. 비동기 다운로드 적용 ( + )


# 비동기 다운로드 예시
  (, )
      .()  
          .()  
             (, '')  
                    ..()
                    .()

**해결책**
. 키워드 개수 증가 개 → 개
