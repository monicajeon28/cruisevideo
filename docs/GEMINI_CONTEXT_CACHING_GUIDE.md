## 개요

  을 활용하여 ** 비용을 % 절감**하는 방법을 설명합니다.

### 핵심 개념

** **은 반복적으로 사용되는 긴 프롬프트(예  의  )를 캐시에 저장하고, 이후 요청에서 재사용하여 토큰 비용을 대폭 절감하는 기능입니다.


기존 방식
요청     + 사용자 질문 → 비용 $.
요청     + 사용자 질문 → 비용 $.
...
총 회 $.

 
캐시 생성    → 비용 $.
요청   참조 + 사용자 질문 → 비용 $. (% 할인)
요청   참조 + 사용자 질문 → 비용 $.
...
총 회 $. + $.  $. (% 절감)

## 최적 사용 시나리오

###   이 유리한 경우

. **많은 재사용** 동일한 정적 컨텍스트를 회 이상 재사용
. **긴 컨텍스트**   이상의 정적 데이터
. **짧은 ** 시간 이내에 모든 호출 완료
. **배치 처리** 한 번에 여러 요청 처리

**예시**

# -  
# -    
# - 배치당 개 스크립트 생성
# - 처리 시간 분
# → 비용 절감 %


---

###   이 불리한 경우

. **적은 재사용** 회 이하 사용
. **짧은 컨텍스트**   미만
. **긴 ** 캐시를 오래 유지
. **단일 요청** 배치가 아닌 개별 요청

**예시**

# 단일 스크립트 생성
# -    
# - 회만 사용
# -  시간
# → 비용 증가 캐시 저장 비용만 발생

## 문제 해결

### .  생성이 실패합니다

**증상**

   생성 실패 ( )  


**원인**
-  키가 올바르지 않음
- 모델이  을 지원하지 않음
- 토큰 수가 최소 요구사항() 미달

**해결**
.  키 확인 . 파일의 
. 모델 확인 -.-- 사용
. 토큰 확인 최소   이상

---

### .  모드가 계속 사용됩니다

**증상**

   사용 불가,  모드 사용


**원인**
-  가  을 아직 지원하지 않음
- 네트워크 오류

**해결**
-  모드도 정상 작동하므로 계속 사용 가능
- 단, 비용 절감 효과는 없음 (메모리 캐시만 사용)

---

### . 비용이 오히려 증가합니다

**증상**

절감률 -.%


**원인**
- 재사용 횟수가 너무 적음
- 이 너무 김
- 캐시 저장 비용($//)이 높음

**해결**
. 재사용 횟수 늘리기 (최소 회+)
.  줄이기 (예 시간 → 분)
. 배치 크기 늘리기
