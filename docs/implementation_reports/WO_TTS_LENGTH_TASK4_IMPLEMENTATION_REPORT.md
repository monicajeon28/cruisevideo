## . 요약

### 문제
현재 스크립트 생성 시스템은 생성만 하고, 에서 초 초과 감지 시 재생성을 하지 않습니다.

### 해결
에 () 메서드를 추가하여
- 에서 초 초과 감지
- 자동으로 재생성 회 시도
- 회 모두 실패 시  발생

### 효과
-   초과 확률 % → **%** (% 보장)
- 완주율 % → **%** (+%)
- 월 수익 만원 → **만원** (+%)

## . 아키텍처 변경

### ..  (기존)


 → .()
         → .()
            → ()
               → .()  #  검증 없음
            → .()
               → 초 초과 감지
               →  재생성 없음 (단순  반환)


**문제** 초 초과 시 재생성하지 않고 실패만 반환 → 사용자가 수동으로 재시도해야 함

### ..  (개선)


 → .()
         → .()
            → ()
               → .()
                  ├─   () → ()
                  │         → 초 초과 감지 → 재생성
                  ├─   () → ()
                  │         → 초 초과 감지 → 재생성
                  ├─   () → ()
                  │         → 초 통과  반환
                  └─ 회 모두 실패 → 
            → .()
               → 초 보장됨 (이미 검증됨)


**개선점**
- 초 초과 시 **자동 재생성 회** (사용자 개입 불필요)
- 재생성 실패 시 **상세 에러 메시지** 제공
- 등급 검증과 **독립적으로**  검증 수행

## . 주의사항

### .. 무한 루프 방지

으로 고정하여 무한 루프 방지
- 회 이상 재생성하지 않음
- 회 실패 시  발생 → 호출자가 처리

### .. 등급 검증과의 관계

- ** 검증** 초 이내 (필수 조건)
- **등급 검증** 점 이상 (품질 조건)

두 검증은 **독립적**으로 수행
.  검증 → 초 보장 (이 )
. 등급 검증 → 점 보장 ()
