## .  (  )

### -  를 코어 엔진으로 채택

**상태** 승인됨

**컨텍스트**
- 기존 .는 중급 품질
-  는 상위 .% 시스템 (- )
- 그러나 현재 파이프라인에 통합되지 않음

**결정**
 를 썸네일 생성의 코어 엔진으로 채택하고, .는 단계적으로 폐기

**결과**
- 장점
  - 매력도 점수 시스템 활용 가능
  - 스마트 텍스트 배치 (오버랩 감지)
  - / 테스트 변형 생성
  - 카테고리별 템플릿 ( 타겟 포함)

- 단점
  - 통합 작업 필요
  - 기존 코드와의 호환성 고려

- 마이그레이션 전략
  .  레이어 신규 생성
  . .에서   호출
  . .는 폴백으로 유지

### - 로컬 이미지 우선, 스마트 선택 적용

**상태** 승인됨

**컨텍스트**
- 의 랜덤 선택으로 부적절한 이미지 선택 가능
- 품질 기준 없이 모든 이미지가 동일 확률로 선택됨

**결정**
를 신규 개발하여 품질 기반 이미지 선택 구현

**결과**
- 장점
  - 매력도 점수 점 이상 이미지만 사용
  - 중복 방지 (최근 개 제외)
  - 카테고리별 최적 이미지 선택

- 단점
  - 초기  구축 시간 필요
  - 계산 오버헤드 증가 (캐싱으로 완화)

- 구현 방법
  . 로 이미지 메타데이터  구축
  .  의 매력도 점수 시스템 활용
  . 적합성 점수 시스템 추가 개발

**컨텍스트**
- 자동 생성된 썸네일의 품질 검증 필요
- 사람 검수 전 자동 필터링으로 효율성 향상

**결정**
를 통한 자동 품질 검증 시스템 도입

**결과**
- 장점
  - 저품질 썸네일 자동 재생성
  - 사람 검수 부담 감소
  - 일관된 품질 기준 적용

- 단점
  - 검증 로직 개발 필요
  - 재생성 시 시간 증가 가능

- 품질 게이트 기준
  - 최소 점수 점
  - 필수 해상도, 텍스트 가독성, 색상 대비
  - 재시도 최대 회
  - 실패 시 사람 검수 플래그

### - 수동 큐레이션 + 자동화 하이브리드 접근

**상태** 승인됨

**컨텍스트**
- 완전 자동화는 초기 품질 문제 발생 가능
- 완전 수동은 확장성 부족

**결정**
초기에 고품질 이미지 세트를 수동 큐레이션하고, 이후 자동화 시스템으로 확장

**결과**
-   수동 큐레이션
  - 카테고리별 -개 고품질 이미지 선별
  - 메타데이터 태깅 (카테고리, 태그, 품질 등급)
  - 에 "" 플래그 추가

-   자동화 확대
  - 큐레이션된 이미지 우선 사용
  - 부족 시 자동 선택 (품질 필터 적용)
  - 사용 후 피드백 반영

- 장기 전략
  -  모델 학습용 데이터 수집
  - 자동 큐레이션 정확도 향상
  - 사람 개입 최소화

이 아키텍처 설계서는  프로젝트의 썸네일 생성 시스템을 "상위 .% 프로덕션급"으로 업그레이드하기 위한 청사진을 제공합니다.

**핵심 전략**

. **  활용** 이미 보유한 상위 .% 시스템을 파이프라인에 통합
. **스마트 이미지 선택** 랜덤 선택에서 품질 기반 선택으로 전환

. **하이브리드 접근** 수동 큐레이션 + 자동화로 점진적 개선
. **레이어 아키텍처** 명확한 책임 분리로 유지보수성 향상

**기대 효과**

- 썸네일 품질 % 향상
- 부적절한 이미지 선택 % 감소
- 사람 검수 시간 % 절감
- 일관된 브랜드 이미지 구축
