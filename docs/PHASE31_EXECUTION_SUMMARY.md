## 가지 문제 상세 분석

### .  개 누락

**문제**
- //. 파일 미존재
- . 실행 시 

**근본 원인**
-  에서 ,  템플릿만  변환
-    부재 → 누락 발견 실패

**해결 (-, 시간)**
- 원본 파일()을 복사 후 변수 치환
- 곳 변경 (예 "만원" → "")

**영향**
- 카테고리 개 중 개 사용 불가 (, , )

---

### . / 중복

**문제**
- 일부 템플릿에서  사용
- .에  변수 미정의 → 

**근본 원인**
- 하향식 설계 ( →  마이그레이션) + 역호환성 검증 누락
-  (기존)   (신규) 통합 실패

**해결 (-, 분 + , 시간)**
- **즉시**  변수 추가 (와 동일 값)
- **장기**  에서  제거, 로 통일

**영향**
- 중복 변수 유지 비용 (코드 복잡도 증가)
- 혼란 (어느 변수를 써야 하는지 불명확)

---

### .  미정의

**문제**
-  템플릿에서  사용
- .에 변수 미정의 → 

**근본 원인**
- 수동 변수 정의 방식 (템플릿 스캔 자동화 부재)
- 복잡도 예측 실패 ( 섹션에 , 만 정의)

**해결 (-, 분 + -, 분)**
- **즉시**  변수 추가 ( "일 이내")
- **장기** 템플릿 자동 스캔 스크립트 (.)

**영향**
-  프리미엄 긴급성 표시 불가 ("주 이내" 대신 "일 이내" 사용)

---

### .  미정의

**문제**
- // 템플릿에서  사용
- .에 변수 미정의 → 

**근본 원인**
-  섹션에 만 정의 ( 누락)
-   미수행

**해결 (-, 분)**
- **즉시**  변수 추가
- 개 조합별 다른 표현
  -  "시간 한국어 케어"
  -  "시간 한국어 케어 + 여행자 보험"
  -  "시간 전담 컨시어지 +  케어"

**영향**
- 한국어 케어 서비스 강조 불가 ( 타겟 핵심 요소)

---

### .  형식 오류

**문제**
- .에서 문자열 형식 사용
  
   ", "  #  문자열
  
- .에서 배열 기대
  
        # 
  

**근본 원인**
-  스키마 검증 부재 (  미사용)
-  위반 (.이 가지 책임)

**해결 (-, 분)**
- **즉시** 개 카테고리 를 배열로 수정
  
   "", ""  #  배열
  

**영향**
- 렌더링 실패 () → 영상 생성 불가

---

### .  미활용

**문제**
- .에  정의
  
   "", ".", "."
  
- 하지만 .에서 이 필드 무시 → 전체 치환

**근본 원인**
-  미적용 (테스트 없이 코드 작성)
- 설계와 구현 불일치

**해결 (-, 시간)**
- **즉시** 섹션 필터링 로직 추가 (())
- **장기** 단위 테스트 작성 (섹션 필터링 검증)

**영향**
- 불필요한 섹션도 치환 (. 등)
- 의도하지 않은 변수 노출

---

### .  필드 미활용

**문제**
- .에  필드가 문서화 용도로만 사용
-   미구현 → 개발자가  직접 열어야 함

**근본 원인**
- 문서화 자동화 부재
-  → 자동 문서 생성 연동 실패

**해결 (-, 분)**
- **즉시** . 작성
  - . → .
  -  필드를 "비고" 컬럼에 표시

**영향**
- 접근성 하락 ( 정보 확인 어려움)
- 지식 전이 실패 (신규 개발자 온보딩 시간 증가)

---

### . 캐시 전략 미구현

**문제**
- .에서 매번 . 전체 로드 ()
- 성능 저하 (회 렌더링 시  메모리 사용)

**근본 원인**
-  (  ) 위반
  -  렌더링 시  변수까지 로드
- 캐시 전략 부재

**해결 (, 시간, 선택)**
- **즉시** 싱글톤 패턴 + 메모리 캐시
- **장기** - 분리 (필요한 부분만 로드)
  - . (개 조합만)
  - . (변수 개만)

**영향**
- 성능 배 저하 (캐시 사용 시 . → .)
- 메모리 낭비 (회 렌더링 시 )
