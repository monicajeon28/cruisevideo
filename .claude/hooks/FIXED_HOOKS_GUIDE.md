#  올바른    구현 완료!

##  문제 원인

###  이전에 사용한 잘못된 이벤트 이름


  "" 
    "--" ...,   //  존재하지 않음
    "-" ...,       //  존재하지 않음
    "-" ...           //  존재하지 않음
  



###  올바른 공식 이벤트 이름

 는 **가지  이벤트**를 지원합니다

. **** - 사용자 프롬프트 제출 시
. **** - 도구 실행 전 (차단 가능)
. **** - 도구 실행 후
. **** - 메인 에이전트 완료 시
. **** - 서브에이전트 완료 시
. **** - 세션 시작 시
. **** - 세션 종료 시
. **** - 컨텍스트 압축 전
. **** - 권한 요청 시

##   응답 형식

는 ** 형식**으로 응답해야 합니다



  "" ,              //   차단,   계속
  "" "이유",            // 차단 시 표시할 메시지
  "" "추가 정보",      // 사용자에게 보여줄 피드백
  "" ,     // 출력 억제 여부
  ""              //  에서 계속 여부

##  실제 작동 흐름

### 예시   에이전트 실행


. 사용자 " 만들어줘"
   ↓
.   실행
   → 프롬프트 로그 저장
   ↓
.    호출 ("-")
   ↓
.   실행
   → . - 
   →  - (로그, 환경 체크)
   →  - ( , 스킬 로드)
   →  - (하드코딩 시크릿 탐지)
   →  - (토큰 최적화)
   ↓
. - 에이전트 실행
   ↓
.   실행
   → . - 
   →  - (완료 로그)
   →  - (/ 체크)
   →    (품질 점수 계산)
   ↓
.   실행
   → 최종 종료 로그
   ↓
. 완료

##  지금부터 자동 작동!

이제 ** 을 사용할 때마다** 자동으로 이 실행됩니다!

### 테스트 방법

. **아무 서브에이전트 실행**

"- 에이전트로 //  엔드포인트 만들어줘"


. **로그 확인**

# - 로그
 .//-*.

# - 로그
 .//-*.

# 전역 로그
 .//.

#   점수
 .//-/-*.

# 리소스 관리 (인 경우)
 .///*.

##  자동으로 실행되는 기능들

### .   에이전트
-    자동 로드
-  에이전트별 맞춤 섹션 선택
-  도메인 품질 검증

### .  
-   할당량 체크
-  /메모리 리소스 관리
-  배치 크기 동적 조절
-  시간마다 학습 사이클 트리거

### .  에이전트 (-, - 등)
-    자동 로드
-  관련 스킬 자동 로드
-   타입 체크
-   자동 실행
-  품질 점수 계산 (점 기준)

### .  에이전트
-  환경 변수 체크
-   확인
-   스키마 검증
-  위험 작업 경고

### .  (모든 / 작업)
-  하드코딩된 시크릿 탐지
-    패턴 체크
-  .  확인
