# - 

## 개념

**-**  구현 전에  스펙을 먼저 정의하고, 이를 기반으로 코드 생성


┌─────────────────────────────────────────────────────────────────┐
│                    -                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   .   정의 (/)                              │
│          ↓                                                      │
│   . 스펙 리뷰 ( +  합의)                        │
│          ↓                                                      │
│   .  자동 생성 (--)                    │
│          ↓                                                      │
│   .   / 구현                        │
│          ↓                                                      │
│   .  생성된 타입으로   구현                  │
│          ↓                                                      │
│   .  로  문서화 자동 완성                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘


## 장점

 장점  설명 
------------
 **동기화**  / 간 타입 불일치 제거 
 **병렬 개발**  스펙 합의 후 / 동시 개발 가능 
 **문서화**  코드와 문서가 항상 일치 
 **테스트**  스펙 기반 자동 테스트 생성 가능 
 **타입 안전성**  -- 타입 보장
