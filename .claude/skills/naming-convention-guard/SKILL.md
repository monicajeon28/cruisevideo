#     .

**네이밍 수호자** - 일관되고 의미 있는 이름으로 코드 가독성 향상

## 핵심 철학



  원칙 "이름만 보고 역할을 알 수 있어야 한다"
  목표 "개월 후에도 코드를 이해할 수 있는 네이밍"

  좋은이름의조건
    - 의도를 드러낸다 (,  )
    - 검색 가능하다 (으로 찾기 쉬움)
    - 발음 가능하다 (팀원과 대화 가능)
    - 일관성이 있다 (프로젝트 전체)

  나쁜이름패턴
    - 한 글자 (, ,  제외)
    - 의미 없는 줄임말 (, , )
    - 타입만 있는 이름 (, , )
    - 헝가리안 표기법 (, )
    - 부정 조건 (, )
