##  워크플로우에서의  적용



  
    - 요구사항을 테스트 케이스 목록으로 변환
    - 각 케이스의 입력/출력 명확화
   
    ## 테스트 케이스 목록
    .  유효한 이메일로 사용자 생성 →  반환
    .  중복 이메일 → 
    .  잘못된 이메일 형식 → 
    .  빈 이름 → 
    .  최대 길이 이름 → 성공


   "구현 코드 없이 테스트만 먼저 작성"
   "테스트 실행 시 반드시 실패해야 함"
   
    //  올바른 순서 테스트 먼저
    ('', ()  
      ('     ',  ()  
            .(
           '.',
           ' '
        )

        (.).(/---$/)  //  형식
        (.).('.')
        (.).(' ')
      )
    )
    // 이 시점에서  메서드가 없으므로 테스트 실패


   "테스트를 통과하는 최소한의 코드만 작성"
   
    //   불필요한 기능까지 구현
      
       () 
        // 캐싱 추가 (요청 안 함)
        // 로깅 추가 (요청 안 함)
        // 알림 추가 (요청 안 함)
         ..()
      
    
   
    //   테스트 통과에 필요한 것만
      
       ( ) 
         ..()
      
    


   "테스트가 통과하는 상태에서만 리팩토링"
  
    - "  테스트 통과 상태 유지"
    - "  중복 제거"
    - "  명명 개선"
    - "  복잡도 감소"
