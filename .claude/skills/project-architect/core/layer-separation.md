## 각 레이어별 상세 규칙

### .  


책임
  - 사용자 인터페이스 렌더링
  - 사용자 입력 처리
  - 상태 관리 ( 상태만)
  - 라우팅
  
포함항목
  / "라우팅 진입점"
  / "재사용 가능한 "
  / " 로직 추상화"
  / "페이지 레이아웃"
  
금지사항
  -  직접 / 호출
  -  비즈니스 로직 (검증, 계산)
  -  데이터 변환 로직
  -  직접  접근

코드예시
   
    //  만 담당, 로직은 서비스에 위임
     () 
        ,    ()
      
       ()   /
         /
    
    
   
    //  에서 비즈니스 로직 처리
     () 
       ,   ()
      
      (()  
        ('///' + )
          .(  .())
          .(  
            // 비즈니스 로직이 에 있음!
               ..((, )  
               + . * . * ( - .), 
            )
            ( ...,  )
          )
      , )
    


### .  


책임
  - 비즈니스 규칙 구현
  - 데이터 검증
  - 복잡한 계산/변환
  - 여러 데이터 소스 조합
  
포함항목
  / "비즈니스 서비스"
  / "유스케이스 (선택적)"
  / "도메인 모델"
  / "검증 로직"
  
금지사항
  -   컴포넌트 
  -  직접  요청
  -  직접  쿼리
  -  프레임워크 종속 코드

코드예시
   
    //  순수한 비즈니스 로직
      
      (
          ,
          
      ) 
      
       ( )  
        // 비즈니스 검증
         .()
        
        // 비즈니스 계산
           .()
           .(, )
        
        // 데이터 레이어에 위임
         ..(
          ,
            - ,
           ''
        )
      
      
       ( )  
         .((, )  
           + . * ., 
        )
      
    


### .  


책임
  - 데이터 저장/조회
  - 외부  통신
  - 캐싱
  - 데이터 매핑 (외부 → 내부 형식)
  
포함항목
  / "데이터 저장소 추상화"
  / " 클라이언트"
  / " 연결/쿼리"
  / "캐시 관리"
  / "데이터 변환"
  
금지사항
  -  비즈니스 로직 포함
  -   컴포넌트 
  -  도메인 규칙 구현

코드예시
   
    //  데이터 접근만 담당
      
       ( )    
            ..(//$)
         ..(.)
      
      
       ( )  
            ..('/', )
         ..(.)
      
    
    
    //  응답 → 도메인 모델 변환
      
      ( )  
         
           .,
           ..(.),
           .,
           .,
            (.),
        
      
    


### .  


책임
  - 공통 유틸리티 함수
  - 타입 정의
  - 상수 정의
  - 공통  컴포넌트 (의존성 없는)
  
포함항목
  / "헬퍼 함수"
  / "공통 타입"
  / "상수"
  // "기본  (, )"
  
규칙
  -  다른 레이어에서  가능
  -  의존성 최소화
  -  다른   금지
  -  비즈니스 로직 포함 금지

코드예시
  /. 
    //  순수 유틸리티 - 어디서든 사용 가능
      ( ,   '')  
        .('-', 
         '',
        ,
      ).()
    
    
      ( ,   '--')  
      // ...
    


## 레이어 간 통신 규칙


허용된의존방향
  
    - →  (서비스 호출)
    - →  (유틸리티 사용)
    
  
    - →  (저장소 호출)
    - →  (유틸리티 사용)
    
  
    - →  (유틸리티 사용)
    
  
    - → 없음 (최하위 레이어)

금지된의존방향
  -   →  (역방향)
  -   →  (역방향)
  -   → 모든 레이어 (역방향)
  -  순환 참조 ( →  → )
