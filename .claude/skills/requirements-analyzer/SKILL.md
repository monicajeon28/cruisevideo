## 분석 프레임워크

### . 기능 추출 ( )



  아이디어파싱
    입력 "인스타그램 같은 사진 공유 앱을 만들고 싶어"
    출력
      핵심키워드 "사진", "공유", "소셜"
      유사서비스 "", ""
      암묵적기능 "사용자 계정", "피드", "좋아요"
      
  기능목록화
    방법 "동사 + 명사 조합으로 기능 정의"
    예시
      - "사용자가 회원가입한다"
      - "사용자가 사진을 업로드한다"
      - "사용자가 다른 사용자를 팔로우한다"
      - "사용자가 피드를 본다"
      - "사용자가 좋아요를 누른다"
      
  기능분류
    카테고리
      인증 "회원가입", "로그인", "로그아웃"
      콘텐츠 "업로드", "조회", "삭제"
      소셜 "팔로우", "좋아요", "댓글"
      알림 "푸시", "이메일"


### .  분리 ( )



  포함기준
    - 서비스의 핵심 가치를 전달하는 기능
    - 없으면 서비스가 성립하지 않는 기능
    - 사용자가 가장 먼저 필요로 하는 기능
    
  제외기준
    - "있으면 좋지만" 수준의 기능
    - 복잡도가 높은 기능
    - 다른 기능에 의존하는 고급 기능
    
  
    
      정의 "없으면 서비스 불가능"
      예시 "로그인, 핵심 "
      
    
      정의 " 직후 필요"
      예시 "프로필 편집, 검색"
      
    
      정의 "있으면 좋음"
      예시 "소셜 로그인, 알림"
      
    
      정의 "나중에 고려"
      예시 " 추천, 다국어 지원"


### . 우선순위 매트릭스 ( )



  공식 "우선순위  (중요도  .) + (긴급도  .) + (난이도역수  .)"
  
  중요도기준
    점 "핵심 가치, 없으면 서비스 불가"
    점 "중요한 사용자 경험"
    점 "편의 기능"
    점 "있으면 좋은 기능"
    점 "부가 기능"
    
  긴급도기준
    점 " 필수"
    점 "차 릴리즈 필요"
    점 "차 릴리즈"
    점 "추후 고려"
    점 "언젠가"
    
  난이도기준
    점 "-시간 (쉬움)"
    점 "반나절 (보통)"
    점 "-일 (어려움)"
    점 "주일 (복잡)"
    점 "주일+ (매우 복잡)"


   기능  중요도  긴급도  난이도  점수  순위 
  ------------------------------------------

   게시글         .   
   댓글        .   


### . 의존성 그래프 ( )



  규칙
    - "를 하려면 가 먼저 있어야 한다"
    - "순환 의존성 탐지 및 경고"
    
  예시
    기본레이어
      - "사용자 인증 (모든 것의 기반)"
      - "데이터베이스 스키마"
      
    차레이어
      - "프로필 (인증 필요)"
      - "게시글  (인증 필요)"
      
    차레이어
      - "댓글 (게시글 필요)"
      - "좋아요 (게시글 필요)"
      - "팔로우 (프로필 필요)"
      
    차레이어
      - "피드 (팔로우 필요)"
      - "알림 (좋아요, 댓글 필요)"


  
  인증 ──────────────────────────────────┐
    │                                    │
    ├── 프로필 ─── 팔로우 ─── 피드       │
    │                                    │
    └── 게시글 ─┬─ 댓글 ─────┬── 알림    │
               │            │           │
               └─ 좋아요 ───┘           │
                                        │
   스키마 ────────────────────────────┘


### . 복잡도 경고 ( )



  
    - "실시간 채팅" → " 구현 필요, 복잡도 높음"
    - "결제 시스템" → "보안 중요,  연동 필요"
    - "파일 업로드" → "스토리지 비용, 용량 관리 필요"
    - "위치 기반" → " 권한, 지도  비용"
    - "/ 기능" → "모델 학습/추론 인프라 필요"
    
  
    - "소셜 로그인" → "각 플랫폼 개발자 등록 필요"
    - "이메일 발송" → "스팸 필터, 발송 제한"
    - "검색 기능" → "대용량 시 성능 문제"
    - "다국어 지원" → "번역 관리,  조정"

## 요구사항 분석 체크리스트

### 기능 완전성
  핵심 가치가 명확한가
  로 서비스 가치를 전달할 수 있는가
  누락된 필수 기능이 없는가

### 우선순위
  의존성 순서가 맞는가
  복잡한 기능이 에 포함되지 않았는가
  각 기능의 난이도가 현실적인가

### 실현 가능성
  팀 역량으로 구현 가능한가
  예상 기간이 현실적인가
  기술적 제약이 파악되었는가
