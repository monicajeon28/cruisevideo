#    

## 역할
  형식의 커밋 메시지 작성 및 커밋 수행

## 사용할 
- - - 변경사항 분석

##   형식


타입
   "새 기능 추가"
   "버그 수정"
   "문서 변경"
   "포맷팅 (코드 변경 없음)"
   "리팩토링"
   "테스트 추가/수정"
   "빌드/설정 변경"
   "성능 개선"
   "/ 변경"

형식
  기본 " 설명"
  스코프 "() 설명"
  브레이킹 "! 설명"

예시
  - " 사용자 로그인 기능 추가"
  - "() 토큰 만료 오류 수정"
  - "  설치 가이드 업데이트"
  - "() 사용자 서비스 구조 개선"


## 커밋 워크플로우


변경사항확인
   " "
   "변경된 파일 목록"

스테이징
  선택적 "  파일"
  전체 "  ."
  패치 "  -"

메시지작성
  분석 "변경사항 내용 파악"
  생성 "  형식"

커밋
   "  - '메시지'"

검증
   "  -"


## 커밋 메시지 작성 규칙


제목
  - "자 이내"
  - "명령형 어조 (, , )"
  - "마침표 없음"
  - "첫 글자 대문자"

본문
  - "제목과 빈 줄로 구분"
  - "자 줄바꿈"
  - "무엇을, 왜 변경했는지"
  - "어떻게는 코드가 설명"

푸터
  - " "
  - "이슈 참조 ( #)"


## 자동 커밋 메시지 생성


// 변경사항 분석 후 메시지 생성
 ( ) 
  // 파일 유형별 분류
     ()

  // 주요 변경 파악
     ()

  // 메시지 생성
   ()


// 예시 출력
// 파일 //., ///. 변경
// 생성 "() 사용자 프로필 업데이트 기능 추가"


## 출력 형식


##  커밋 준비

### 변경사항 요약

 상태  파일  변경 
------------------
   //.  + - 
   ///.  + - 
   //.  + 

### 제안 커밋 메시지


() 사용자 프로필 업데이트 기능 추가

- 프로필 이미지 업로드 기능 구현
- 이메일 변경 검증 로직 추가
- 사용자 타입 정의 분리

 #


### 커밋 명령


# 전체 스테이징 후 커밋
  . &&   - "$( ''
() 사용자 프로필 업데이트 기능 추가

- 프로필 이미지 업로드 기능 구현
- 이메일 변경 검증 로직 추가
- 사용자 타입 정의 분리

 #

)"


### 대안 커밋 (분리)

더 세분화된 커밋을 원하시면


# 커밋  타입 정의
  //.
  - "() 사용자 타입 정의 분리"

# 커밋  서비스 로직
  //.
  - "() 프로필 업데이트 서비스 구현"

# 커밋   라우트
  ///.
  - "() 프로필 업데이트 엔드포인트 추가"
