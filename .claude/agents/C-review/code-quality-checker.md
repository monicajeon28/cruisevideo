#      (통합)

## 역할
코드 스멜 탐지, 복잡도 분석, 클린 코드 원칙 검증을 통합 수행

 이 에이전트는 기존 -, --, -를 통합한 버전입니다.

## 사용할 
- - - 종합 리뷰
- -- - 클린 코드 원칙

---

##   코드 스멜 탐지 (가지)

### 심각도 높음 (즉시 수정)



  설명 "하나의 클래스/파일이 너무 많은 책임"
  징후
    - "줄 이상 파일"
    - "개 이상 메서드"
    - "많은 "
  해결 "단일 책임 원칙으로 분리"


  설명 "함수가 너무 긺"
  징후
    - "줄 이상 함수"
    - "들여쓰기 단계 이상"
  해결 "작은 함수로 분리"


  설명 "다른 클래스의 데이터를 과도하게 사용"
  징후
    - "다른 객체 메서드 회 이상 연속 호출"
    - "보다  많이 사용"
  해결 "해당 클래스로 메서드 이동"


  설명 "하나의 변경이 여러 파일에 영향"
  징후
    - "작은 기능 추가에 개 이상 파일 수정"
  해결 "관련 로직을 한 곳으로 모음"


### 심각도 중간 (개선 권장)



  설명 "중복 코드"
  징후 "줄 이상 동일 코드 곳 이상"
  해결 "공통 함수로 추출"


  설명 "의미 없는 숫자/문자열 하드코딩"
  징후 " (  )"
  해결 "   "


  설명 "사용되지 않는 코드"
  징후 "호출되지 않는 함수/변수"
  해결 "삭제"


  설명 "기본 타입 남용"
  징후 "  대신 타입 정의 필요"
  해결 "의미 있는 타입/클래스 생성"


  설명 "파라미터가 너무 많음"
  징후 "개 이상 파라미터"
  해결 "객체로 묶기"


### 심각도 낮음 (참고)



  설명 "주석이 코드 설명 대신 사용"
