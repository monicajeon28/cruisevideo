## 의존성 규칙


허용된방향
  - "/ → /"
  - "/ → /"
  - "/ → /"
  - "/ → /"
  - "모든 곳 → /"

금지된방향
  - "/ → /"
  - "/ → /"

  - "순환 참조"

   ""

## 모듈 구조 권장


/
├── /              #  타입만 (의존성 없음)
│   ├── .
│   └── .
│
├── /                #  순수 로직
│   ├── /          # 유틸리티 (의존성 최소)
│   │   └── .
│   └── /       # 비즈니스 로직
│       ├── / # 서비스 인터페이스
│       ├── .
│       └── .
│
├── /         #   컴포넌트
│   ├── /             # 기본  (만 의존)
│   └── /       # 기능 컴포넌트
│
└── /                #  페이지 (모든 것 사용 가능)
    └── /            #  라우트

### 요약
-  정상 개 파일
-  위반 개 파일
-  순환 참조 건

### 위반 사항

####  역방향 의존성
**파일** //.

     '//'

**문제**  →  방향 금지
**해결** 를 /로 이동

####  순환 참조 감지

//.
    ↓ 
//.
    ↓ 
//.  ← 순환!

**해결** 공통 로직을 별도 모듈로 추출

### 의존성 그래프

/.
  └── /.
        └── //.
              └── //.
                    └── /.



## 자동 수정 제안


역방향해결
  문제 "에서  "
  해결
    - "해당 코드를 로 이동"
    - "또는 인터페이스로 분리"

순환해결
  문제 " ↔  순환"
  해결
    - "공통 부분 추출 →  생성"
    - " → ,  → "
